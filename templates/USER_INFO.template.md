# User Info — Template & Guide

> This is a reference file. Fill in `USER_INFO.md`, not this file.
> All fields are optional. Write as much or as little as is useful.
> You may add fields that are not listed here — the orchestrator will read them.

---

## Identity

**Role**
Your professional role or title. Not limited to any category — write whatever describes what you actually do.
```
Examples:
  UX Designer
  Senior Backend Engineer
  Marketing Consultant
  Corporate Lawyer
  Sales Manager
  Founder & CEO
  Data Analyst
  Product Manager
  Copywriter
  Financial Advisor
```

**Industry**
The sector or domain you work in.
```
Examples:
  EdTech
  FinTech
  Healthcare
  E-commerce
  Legal services
  B2B SaaS
  Media & Publishing
  Real estate
```

**Company / Organization**
Where you work — a company name, your own business name, or a description if you prefer not to name it.
```
Examples:
  Acme Corp
  Freelance / Independent
  My own agency (design studio)
  [Stealth startup]
```

**Experience level**
How you'd describe your level in your primary field.
```
Options (use freely, these are not strict):
  Junior — early career, building foundations
  Mid — independent practitioner
  Senior — deep expertise, handles complex cases
  Lead / Principal — sets direction, guides others
  Founder — running the business
  Other — describe in your own words
```

## Competencies

**Primary skills**
What you do well and regularly. These directly influence which plugins the orchestrator selects.
```
Examples:
  Conversion copywriting, email marketing, A/B testing
  React, TypeScript, REST API design
  User research, usability testing, Figma
  Financial modelling, Excel, PowerPoint
```

**Secondary skills**
What you can do but is not your core focus. Helpful context for edge cases.
```
Examples:
  Basic SQL, some Python scripting
  Light design work in Canva
  Project management, stakeholder communication
```

**Tools**
Platforms, software, environments you work in regularly.
```
Examples:
  Figma, Notion, Linear
  VS Code, GitHub, Docker, AWS
  HubSpot, Salesforce, Klaviyo
  Adobe Illustrator, Webflow
  Google Workspace, Slack
```

## Working Context

**Works as**
Your employment or engagement model.
```
Options:
  Employed — working for a company as a staff member
  Freelance — independent contractor working with multiple clients
  Founder — running your own business
  Consultant — advisory or project-based engagements
  Other — describe in your own words
```

**Team size**
The team you work within day to day.
```
Examples:
  Solo — working alone
  Small team (2–5 people)
  Mid-size team (6–20 people)
  Large organisation (20+ people)
```

**Typical tasks**
What you work on most often. The more specific, the better the plugin match.
```
Examples:
  Writing landing page copy and email sequences
  Building and reviewing pull requests, code review
  Running user interviews and synthesising research findings
  Preparing pitch decks and investor materials
  Managing ad campaigns across Meta and Google
```

## Preferences

**Output style**
How you prefer to receive results from the agent.
```
Examples:
  Detailed with full explanations
  Concise — bullet points and summaries only
  Structured documents ready to use
  Draft first, refine together
```

**Language**
The language you want the orchestrator to communicate in.
```
Examples:
  English
  Russian
  Spanish
  German
```

**Constraints**
Anything the agent should always keep in mind when working with you.
```
Examples:
  Never use jargon — outputs must be readable by non-technical stakeholders
  All output must be GDPR-compliant
  Tone must always remain formal and professional
  No use of AI-generated stock phrases ("delve", "leverage", etc.)
  Output must fit into a Notion template structure
```
